## What This Is

**simdfix** is a SIMD-accelerated FIX (Financial Information Exchange) protocol parser in C++23, targeting ARM NEON. It parses raw FIX message bytes using 16-byte parallel NEON operations and SWAR (SIMD Within A Register) digit parsing.

## Architecture

### Data Flow

```
Raw FIX bytes
    тЖТ Decoder::parse()       # SIMD tokenization; fills Token[64] array
    тЖТ MessageHandler::handle() # Dispatch by MsgType tag
    тЖТ LogonDecoder / LogoutDecoder  # Field access via LinearSearch
    тЖТ Application handler
```

### Key Components

**`Decoder.hpp`** тАФ Core engine. Processes 16 bytes/cycle with NEON to detect `=` (tag end) and `0x01` (field end). Returns a flat `Token[]` array (position + tag + length, no copies) and a `ParserStatus`. Handles fragmented messages and split tags across chunk boundaries.

**`Uint8x16.hpp`** тАФ Thin NEON wrapper (`uint8x16_t`). All SIMD operations go through here. If porting off ARM, this is the only layer to replace.

**`MessageDecoder.hpp`** тАФ Generic field extractor templated on a Protocol (e.g., `Logon`, `Logout`). Uses `LinearSearch` to find a tag in the token array and returns `std::expected<T, ParserStatus>`. Protocol must expose a sorted `Tags` array and a `Grammar` array of `Dictionary` structs.

**`Grammar.hpp`** тАФ Compile-time protocol metadata (`constexpr`): tag numbers, types, mandatory flags. Generated by `Generator.cpp` from `protocol.xml`. Do not edit manually.

**`Generator.cpp`** тАФ Reads `protocol.xml` and emits `LogonDecoder.hpp`, `LogoutDecoder.hpp`, `Grammar.hpp`. Run it to regenerate after changing the protocol spec.

**`MessageHandler.hpp`** тАФ Application entry point. Subclass it and override `handle(LogonDecoder&)` / `handle(LogoutDecoder&)`.

### Design Constraints

- **Header-only library** (`INTERFACE` CMake target) тАФ no compiled translation units in the parser.
- **Max 64 tokens** per message тАФ compile-time constant in `Decoder.hpp`.
- **Protocol `Tags` must be sorted** тАФ `MessageDecoder` assumes this for search correctness.
- **ARM NEON only** тАФ `Uint8x16` wraps `arm_neon.h` intrinsics directly.
- **No exceptions** тАФ errors propagate as `std::expected<T, ParserStatus>`.

### Adding a New Message Type

1. Add the message definition to `protocol.xml`.
2. Run `Generator` to regenerate `Grammar.hpp` and produce the new `FooDecoder.hpp`.
3. Add a `handle(FooDecoder&)` override in `MessageHandler.hpp` dispatch and in application handlers.
